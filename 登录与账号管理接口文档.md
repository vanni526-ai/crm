# 登录与账号管理接口文档

## 文档概述

本文档为前端APP开发团队提供登录、账号注销、账号恢复等接口的完整说明。系统实现了账号注销的30天缓冲期机制，用户在注销后30天内可以通过登录恢复账号。

**API基础URL**: `https://crm.bdsm.com.cn/api`

**更新日期**: 2026-02-27

---

## 核心接口列表

| 接口名称 | 接口路径 | 权限要求 | 功能说明 |
|---------|---------|---------|---------|
| 获取当前用户信息 | `auth.me` | 需登录 | 获取当前登录用户的信息 |
| 登出 | `auth.logout` | 需登录 | 退出登录 |
| 账号恢复 | `auth.restoreAccount` | 公开 | 恢复注销中的账号 |
| 查询注销状态 | `auth.getDeletionStatus` | 公开 | 查询账号的注销状态 |

## 1. 登录拦截机制（isDeleted检查）

### 机制说明

系统在用户登录时自动检查账号的注销状态（`isDeleted`字段），如果账号处于注销中状态，将阻止登录并返回结构化错误信息，提示用户恢复账号。

### 注销状态说明

| 状态值 | 状态名称 | 说明 | 是否可登录 |
|-------|---------|------|-----------|
| `0` | 正常 | 账号正常使用 | ✅ 可以 |
| `1` | 注销中 | 账号已发起注销，30天缓冲期内 | ❌ 不可以（可恢复） |
| `2` | 已脱敏 | 账号超过30天，数据已脱敏 | ❌ 不可以（不可恢复） |

### 登录拦截错误响应

当用户尝试登录一个注销中的账号时，系统返回以下结构化错误信息：

```json
{
  "success": false,
  "error": {
    "code": "ACCOUNT_PENDING_DELETION",
    "message": "您的账号正在注销中，30天内可通过登录恢复",
    "data": {
      "userId": 123,
      "phone": "138****0000",
      "daysRemaining": 28,
      "deletedAt": "2026-02-26T10:30:00.000Z",
      "recoveryDeadline": "2026-03-28T10:30:00.000Z"
    }
  }
}
```

### 错误字段说明

| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|--------|
| `code` | string | 错误代码 | `"ACCOUNT_PENDING_DELETION"` |
| `message` | string | 错误消息 | `"您的账号正在注销中，30天内可通过登录恢复"` |
| `data.userId` | number | 用户ID | `123` |
| `data.phone` | string | 脱敏后的手机号 | `"138****0000"` |
| `data.daysRemaining` | number | 剩余可恢复天数 | `28` |
| `data.deletedAt` | string | 注销发起时间（ISO 8601） | `"2026-02-26T10:30:00.000Z"` |
| `data.recoveryDeadline` | string | 恢复截止时间（ISO 8601） | `"2026-03-28T10:30:00.000Z"` |

### 前端处理示例（React Native）

```typescript
import { trpc } from '@/lib/trpc';
import { Alert } from 'react-native';

function LoginScreen() {
  const loginMutation = trpc.auth.login.useMutation({
    onError: (error) => {
      // 检查是否是账号注销错误
      if (error.data?.code === 'ACCOUNT_PENDING_DELETION') {
        const errorData = error.data.data;
        
        // 显示恢复账号对话框
        Alert.alert(
          '账号正在注销中',
          `您的账号已于${formatDate(errorData.deletedAt)}发起注销，还有${errorData.daysRemaining}天可以恢复。是否立即恢复账号？`,
          [
            {
              text: '取消',
              style: 'cancel'
            },
            {
              text: '恢复账号',
              onPress: () => {
                // 跳转到账号恢复页面
                navigation.navigate('AccountRecovery', {
                  userId: errorData.userId,
                  phone: errorData.phone,
                  daysRemaining: errorData.daysRemaining
                });
              }
            }
          ]
        );
      } else {
        // 其他登录错误
        Alert.alert('登录失败', error.message);
      }
    },
    onSuccess: (data) => {
      // 登录成功，跳转到首页
      navigation.replace('Home');
    }
  });

  const handleLogin = (username: string, password: string) => {
    loginMutation.mutate({ username, password });
  };

  return (
    // ... 登录表单UI
  );
}
```

### 接口信息

- **请求方法**: `query`
- **权限要求**: 需登录

### 请求参数

### 前端调用示例

```typescript
import { trpc } from '@/lib/trpc';

function ProfileScreen() {
  const { data, isLoading, error } = trpc.auth.me.useQuery();

  if (isLoading) return <Text>加载中...</Text>;
  if (error) return <Text>加载失败: {error.message}</Text>;

        <View style={styles.memberBadge}>
          <Text>会员有效期至：{formatDate(user.membershipExpiresAt)}</Text>
        </View>
      )}
    </View>
  );
}
```

## 3. 登出

### 接口信息

- **接口路径**: `auth.logout`
- **请求方法**: `mutation`
- **权限要求**: 需登录

### 请求参数

无需参数

### 响应示例

```json
{
```

```typescript
import { trpc } from '@/lib/trpc';

function ProfileScreen() {
  const logoutMutation = trpc.auth.logout.useMutation({
```

## 4. 账号恢复

### 接口信息

- **接口路径**: `auth.restoreAccount`
- **请求方法**: `mutation`
- **权限要求**: 公开接口，无需登录

### 功能说明

用户在账号注销后的30天内，可以通过此接口恢复账号。恢复后，账号状态恢复正常，可以正常登录使用。

### 请求参数

```typescript
{
  userId: number;  // 必填：用户ID
}
```

### 响应示例

**成功响应**:
```json
{
  "success": true,
  "message": "账号恢复成功，您现在可以正常登录了"
}
```

**失败响应（账号不存在）**:
```json
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "账号不存在"
  }
}
```

**失败响应（账号未注销）**:
```json
{
  "success": false,
  "error": {
    "code": "BAD_REQUEST",
    "message": "账号未处于注销状态，无需恢复"
  }
}
```

**失败响应（超过30天）**:
```json
{
  "success": false,
  "error": {
    "code": "FORBIDDEN",
    "message": "账号已超过30天恢复期限，无法恢复"
  }
}
```

### 前端调用示例

```typescript
import { trpc } from '@/lib/trpc';

function AccountRecoveryScreen({ route }) {
  const { userId, phone, daysRemaining } = route.params;
  
  const restoreMutation = trpc.auth.restoreAccount.useMutation({
    onSuccess: () => {
      Alert.alert(
        '恢复成功',
        '您的账号已成功恢复，现在可以正常登录了',
        [
          {
            text: '立即登录',
            onPress: () => navigation.replace('Login')
          }
        ]
      );
    },
    onError: (error) => {
      Alert.alert('恢复失败', error.message);
    }
  });

  const handleRestore = () => {
    Alert.alert(
      '确认恢复账号',
      `您确定要恢复账号吗？恢复后可以正常使用所有功能。`,
      [
        { text: '取消', style: 'cancel' },
        { text: '确定', onPress: () => restoreMutation.mutate({ userId }) }
      ]
    );
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>账号恢复</Text>
      <Text style={styles.info}>手机号：{phone}</Text>
      <Text style={styles.info}>剩余可恢复天数：{daysRemaining}天</Text>
      <Button 
        title="恢复账号" 
        onPress={handleRestore}
        loading={restoreMutation.isLoading}
      />
      <Button 
        title="返回登录" 
        onPress={() => navigation.replace('Login')}
        type="outline"
      />
    </View>
  );
}
```

## 5. 查询注销状态

### 接口信息

- **接口路径**: `auth.getDeletionStatus`
- **请求方法**: `query`
- **权限要求**: 公开接口，无需登录

### 功能说明

查询指定用户的账号注销状态，包括注销时间、剩余天数等信息。

### 请求参数

```typescript
{
  userId: number;  // 必填：用户ID
}
```

### 响应字段说明

| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|--------|
| `userId` | number | 用户ID | `123` |
| `isDeleted` | number | 注销状态（0=正常，1=注销中，2=已脱敏） | `1` |
| `deletedAt` | string \| null | 注销发起时间 | `"2026-02-26T10:30:00.000Z"` |
| `daysRemaining` | number \| null | 剩余可恢复天数 | `28` |
| `recoveryDeadline` | string \| null | 恢复截止时间 | `"2026-03-28T10:30:00.000Z"` |
| `canRestore` | boolean | 是否可以恢复 | `true` |

### 响应示例

**正常账号**:
```json
{
  "success": true,
  "data": {
    "userId": 123,
    "isDeleted": 0,
    "deletedAt": null,
    "daysRemaining": null,
    "recoveryDeadline": null,
    "canRestore": false
  }
}
```

**注销中的账号**:
```json
{
  "success": true,
  "data": {
    "userId": 123,
    "isDeleted": 1,
    "deletedAt": "2026-02-26T10:30:00.000Z",
    "daysRemaining": 28,
    "recoveryDeadline": "2026-03-28T10:30:00.000Z",
    "canRestore": true
  }
}
```

**已脱敏的账号**:
```json
{
  "success": true,
  "data": {
    "userId": 123,
    "isDeleted": 2,
    "deletedAt": "2026-01-26T10:30:00.000Z",
    "daysRemaining": 0,
    "recoveryDeadline": "2026-02-25T10:30:00.000Z",
    "canRestore": false
  }
}
```

### 前端调用示例

```typescript
import { trpc } from '@/lib/trpc';

function AccountStatusScreen({ route }) {
  const { userId } = route.params;
  
  const { data, isLoading, error } = trpc.auth.getDeletionStatus.useQuery({
    userId
  });

  if (isLoading) return <Text>查询中...</Text>;
  if (error) return <Text>查询失败: {error.message}</Text>;

  const status = data?.data;

  return (
    <View style={styles.container}>
      <Text style={styles.title}>账号状态</Text>
      {status.isDeleted === 0 && (
        <Text style={styles.status}>账号正常</Text>
      )}
      {status.isDeleted === 1 && (
        <View>
          <Text style={styles.status}>账号注销中</Text>
          <Text>注销时间：{formatDate(status.deletedAt)}</Text>
          <Text>剩余可恢复天数：{status.daysRemaining}天</Text>
          <Text>恢复截止时间：{formatDate(status.recoveryDeadline)}</Text>
          {status.canRestore && (
            <Button 
              title="恢复账号" 
              onPress={() => navigation.navigate('AccountRecovery', { userId })}
            />
          )}
        </View>
      )}
      {status.isDeleted === 2 && (
        <View>
          <Text style={styles.status}>账号已永久删除</Text>
          <Text>注销时间：{formatDate(status.deletedAt)}</Text>
          <Text>该账号已超过30天恢复期限，无法恢复</Text>
        </View>
      )}
    </View>
  );
}
```

## 账号注销业务流程

### 完整流程图

```
用户登录
    ↓
检查isDeleted状态
    ↓
┌─────────────────────┐
│ isDeleted = 0       │ → 登录成功，正常使用
│ （账号正常）         │
└─────────────────────┘
    ↓
┌─────────────────────┐
│ isDeleted = 1       │ → 阻止登录，返回错误信息
│ （注销中，30天内）   │    ↓
└─────────────────────┘   显示恢复提示
    ↓                      ↓
┌─────────────────────┐   用户选择恢复
│ isDeleted = 2       │    ↓
│ （已脱敏，超过30天） │   调用restoreAccount接口
└─────────────────────┘    ↓
    ↓                    账号恢复成功
阻止登录，提示账号已删除    ↓
                        isDeleted = 0
                          ↓
                        可以正常登录
```

### 时间线说明

1. **T+0天**: 用户发起账号注销，`isDeleted`设置为1，`deletedAt`记录当前时间
2. **T+1至T+29天**: 账号处于注销中状态，用户尝试登录时被拦截，可以选择恢复账号
3. **T+30天**: 定时任务自动执行数据脱敏，`isDeleted`设置为2，`anonymizedAt`记录脱敏时间
4. **T+30天后**: 账号永久删除，无法恢复

## 错误处理

所有接口在发生错误时返回以下格式：

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误消息"
  }
}
```

### 常见错误码

| 错误码 | 说明 | 触发场景 |
|-------|------|---------|
| `ACCOUNT_PENDING_DELETION` | 账号正在注销中 | 登录时账号处于注销状态 |
| `UNAUTHORIZED` | 未登录或Token无效 | 访问需要登录的接口时未提供有效Token |
| `NOT_FOUND` | 用户不存在 | 查询或恢复不存在的用户 |
| `BAD_REQUEST` | 请求参数错误 | 参数格式错误或缺少必填参数 |
| `FORBIDDEN` | 操作被禁止 | 尝试恢复超过30天的账号 |
| `INTERNAL_SERVER_ERROR` | 服务器内部错误 | 服务器异常 |

```typescript
import { createTRPCReact } from '@trpc/react-query';
import { httpBatchLink } from '@trpc/client';

export const trpc = createTRPCReact<AppRouter>();

export const trpcClient = trpc.createClient({

      },
    }),
  ],
```

### 全局错误处理

```typescript
// lib/errorHandler.ts
import { TRPCClientError } from '@trpc/client';
import { Alert } from 'react-native';

export function handleTRPCError(error: TRPCClientError<any>, navigation: any) {
  // 处理账号注销错误
  if (error.data?.code === 'ACCOUNT_PENDING_DELETION') {
    const errorData = error.data.data;
    Alert.alert(
      '账号正在注销中',
      `您的账号已于${formatDate(errorData.deletedAt)}发起注销，还有${errorData.daysRemaining}天可以恢复。是否立即恢复账号？`,
      [
        { text: '取消', style: 'cancel' },
        {
          text: '恢复账号',
          onPress: () => {
            navigation.navigate('AccountRecovery', {
              userId: errorData.userId,
              phone: errorData.phone,
              daysRemaining: errorData.daysRemaining
            });
          }
        }
      ]
    );
    return;
  }

  // 处理未登录错误
  if (error.data?.code === 'UNAUTHORIZED') {
    clearAuthToken();
    navigation.replace('Login');
    return;
  }

  // 其他错误
  Alert.alert('错误', error.message);
}
```
