**最后更新**: 2026-02-12

## 概述

合伙人管理模块提供了完整的合伙人信息管理、费用承担配置、城市分配、合同管理、分红记录等功能的接口。所有接口基于tRPC协议，使用TypeScript类型安全，支持端到端的类型推导。

**基础路径**: `/api/trpc/partnerManagement`  
**认证方式**: 所有接口均需要用户登录认证（protectedProcedure）

---

## 接口列表

### 1. 合伙人基础管理

#### 1.1 获取所有合伙人列表

**接口名称**: `partnerManagement.list`  
**请求类型**: Query  
**功能描述**: 获取所有合伙人的列表，支持按激活状态筛选

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| isActive | boolean | 否 | 是否激活（true=激活，false=停用，不传=全部） |

**响应数据**: 返回合伙人数组，每个合伙人包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | number | 合伙人ID |
| userId | number | 关联的用户ID |
| name | string | 合伙人姓名 |
| phone | string | 手机号 |
| idCardNumber | string | 身份证号码 |
| idCardFrontUrl | string | 身份证正面照片URL |
| idCardBackUrl | string | 身份证反面照片URL |
| profitRatio | string | 分红比例（decimal） |
| profitRule | string | 分红规则说明 |
| brandFee | string | 品牌费用（decimal） |
| techServiceFee | string | 技术服务费（decimal） |
| deferredPaymentTotal | string | 后付款总额（decimal） |
| deferredPaymentRule | string | 后付款规则 |
| contractStartDate | Date | 合同开始日期 |
| contractEndDate | Date | 合同结束日期 |
| accountName | string | 开户名 |
| bankName | string | 开户行 |
| accountNumber | string | 银行账号 |
| profitPaymentDay | number | 每月分红支付日（1-31） |
| expenseCoverage | JSON | 费用承担配置对象 |
| isActive | boolean | 是否激活 |
| notes | string | 备注 |
| createdAt | Date | 创建时间 |
| createdBy | number | 创建人ID |

**调用示例**:

```typescript
// 获取所有激活的合伙人
const activePartners = await trpc.partnerManagement.list.useQuery({
  isActive: true
});

// 获取所有合伙人（包括停用的）
const allPartners = await trpc.partnerManagement.list.useQuery();
```

#### 1.3 创建合伙人

**接口名称**: `partnerManagement.create`  
**请求类型**: Mutation  
**功能描述**: 创建新的合伙人记录，可选择关联现有用户或自动创建新用户账号

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | number | 否 | 关联的用户ID（不提供则自动创建） |
| name | string | 是 | 合伙人姓名 |
| phone | string | 否 | 手机号（自动创建用户时必填） |
| idCardNumber | string | 否 | 身份证号码 |
| idCardFrontUrl | string | 否 | 身份证正面照片URL |
| idCardBackUrl | string | 否 | 身份证反面照片URL |
| profitRatio | string | 是 | 分红比例（如"0.50"表示50%） |
| profitRule | string | 否 | 分红规则说明 |
| brandFee | string | 否 | 品牌费用 |
| techServiceFee | string | 否 | 技术服务费 |
| deferredPaymentTotal | string | 否 | 后付款总额 |
| deferredPaymentRule | string | 否 | 后付款规则 |
| contractStartDate | string | 否 | 合同开始日期（ISO 8601格式） |
| contractEndDate | string | 否 | 合同结束日期（ISO 8601格式） |
| accountName | string | 否 | 开户名 |
| bankName | string | 否 | 开户行 |
| accountNumber | string | 否 | 银行账号 |
| notes | string | 否 | 备注 |

**响应数据**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | number | 新创建的合伙人ID |
| userId | number | 关联的用户ID |
| userCreated | boolean | 是否新创建了用户账号 |

**业务逻辑**:
- 如果不提供`userId`，系统会自动创建用户账号，默认密码为`123456`，角色为`user,cityPartner`
- 自动创建用户时，手机号不能与现有用户重复
- 创建的用户openId格式为`partner_{timestamp}_{random}`

**错误码**:
- `BAD_REQUEST`: 创建合伙人时必须提供手机号（自动创建用户时）
- `BAD_REQUEST`: 该手机号已被使用

**调用示例**:

```typescript
// 自动创建用户账号
const result = await trpc.partnerManagement.create.useMutation({
  name: "张三",
  phone: "13800138000",
  profitRatio: "0.50",
  contractStartDate: "2026-01-01",
  contractEndDate: "2028-12-31"
});

// 关联现有用户
const result2 = await trpc.partnerManagement.create.useMutation({
  userId: 10,
  name: "李四",
  profitRatio: "0.60"
});
```

#### 1.4 更新合伙人信息

**接口名称**: `partnerManagement.update`  
**请求类型**: Mutation  
**功能描述**: 更新合伙人的基本信息，支持部分字段更新

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | number | 是 | 合伙人ID |
| name | string | 否 | 合伙人姓名 |
| phone | string | 否 | 手机号（会同步更新users表） |
| idCardNumber | string | 否 | 身份证号码 |
| idCardFrontUrl | string | 否 | 身份证正面照片URL |
| idCardBackUrl | string | 否 | 身份证反面照片URL |
| profitRatio | string | 否 | 分红比例 |
| profitRule | string | 否 | 分红规则说明 |
| brandFee | string | 否 | 品牌费用 |
| techServiceFee | string | 否 | 技术服务费 |
| deferredPaymentTotal | string | 否 | 后付款总额 |
| deferredPaymentRule | string | 否 | 后付款规则 |
| contractStartDate | string | 否 | 合同开始日期 |
| contractEndDate | string | 否 | 合同结束日期 |
| contractHistory | string | 否 | 合同历史记录 |
| accountName | string | 否 | 开户名 |
| bankName | string | 否 | 开户行 |
| accountNumber | string | 否 | 银行账号 |
| isActive | boolean | 否 | 是否激活 |
| notes | string | 否 | 备注 |

**响应数据**:

```typescript
{ success: true }
```

**业务逻辑**:
- 更新手机号时会同步更新关联的users表
- 新手机号不能与其他用户的手机号重复
- 只更新提供的字段，未提供的字段保持不变

**错误码**:
- `NOT_FOUND`: 合伙人不存在
- `BAD_REQUEST`: 该手机号已被其他用户使用

**调用示例**:

```typescript
await trpc.partnerManagement.update.useMutation({
  id: 1,
  phone: "13900139000",
  accountName: "张三",
  bankName: "中国工商银行北京分行",
  accountNumber: "6222021234567890123"
});
```

**功能描述**: 获取合伙人的费用承担配置（哪些费用由合伙人承担）

| utilityFee | boolean | 是否承担水电费 |
| consumablesFee | boolean | 是否承担耗材费 |
| cleaningFee | boolean | 是否承担保洁费 |
| phoneFee | boolean | 是否承担电话费 |
| deferredPayment | boolean | 是否承担后付款 |
| courierFee | boolean | 是否承担快递费 |
| promotionFee | boolean | 是否承担推广费 |
| teacherFee | boolean | 是否承担老师费用 |
| transportFee | boolean | 是否承担交通费 |
| otherFee | boolean | 是否承担其他费用 |

### 5. 城市管理

#### 5.1 为合伙人分配城市

**接口名称**: `partnerManagement.assignCities`  
**请求类型**: Mutation  
**功能描述**: 为合伙人分配管理的城市（会先删除现有关联，再添加新关联）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| partnerId | number | 是 | 合伙人ID |
| cityIds | number[] | 是 | 城市ID数组 |

**响应数据**:

```typescript
{ success: true }
```

**业务逻辑**:
- 先删除该合伙人的所有城市关联
- 再添加新的城市关联
- 如果`cityIds`为空数组，则清空所有城市关联

**调用示例**:

```typescript
// 分配城市
await trpc.partnerManagement.assignCities.useMutation({
  partnerId: 1,
  cityIds: [1, 2, 3]
});

// 清空所有城市
await trpc.partnerManagement.assignCities.useMutation({
  partnerId: 1,
  cityIds: []
});
```

## 数据类型说明

### Decimal字段处理

所有金额、比例等decimal类型字段在数据库中存储为字符串，在接口传输时也使用字符串格式，以避免浮点数精度问题。

**示例**:
```typescript
// ✅ 正确
profitRatio: "0.50"  // 表示50%
amount: "15000.00"   // 表示15000元

// ❌ 错误
profitRatio: 0.5     // 不要使用number类型
amount: 15000        // 不要使用number类型
```

### 日期字段处理

日期字段支持ISO 8601格式字符串（如`2026-01-01`或`2026-01-01T00:00:00Z`），在数据库中存储为Date类型。

**示例**:
```typescript
contractStartDate: "2026-01-01"
transferDate: "2026-03-10T08:00:00Z"
```

### 枚举类型

**合同状态（contractStatus）**:
- `draft`: 草稿
- `active`: 生效中
- `expired`: 已过期
- `terminated`: 已终止

**打款方式（transferMethod）**:
- `bank_transfer`: 银行转账
- `alipay`: 支付宝
- `wechat`: 微信

**分红记录状态（status）**:
- `pending`: 待处理
- `completed`: 已完成
- `failed`: 失败

## 最佳实践

### 1. 使用tRPC Hooks

在React组件中使用tRPC提供的hooks，自动处理loading、error状态：

```typescript
function PartnerList() {
  const { data, isLoading, error } = trpc.partnerManagement.list.useQuery({
    isActive: true
  });
  
  if (isLoading) return <div>加载中...</div>;
  if (error) return <div>错误: {error.message}</div>;
  
  return (
    <ul>
      {data.map(partner => (
        <li key={partner.id}>{partner.name}</li>
      ))}
    </ul>
  );
}
```

### 2. 使用Mutation处理数据修改

```typescript
function UpdatePartner({ partnerId }) {
  const utils = trpc.useUtils();
  const updateMutation = trpc.partnerManagement.update.useMutation({
    onSuccess: () => {
      // 更新成功后刷新列表
      utils.partnerManagement.list.invalidate();
    }
  });
  
  const handleUpdate = () => {
    updateMutation.mutate({
      id: partnerId,
      phone: "13900139000"
    });
  };
  
  return (
    <button onClick={handleUpdate} disabled={updateMutation.isLoading}>
      {updateMutation.isLoading ? '更新中...' : '更新'}
    </button>
  );
}
```

### 3. 乐观更新（Optimistic Updates）

对于用户体验要求高的场景，可以使用乐观更新：

```typescript
const updateMutation = trpc.partnerManagement.update.useMutation({
  onMutate: async (newData) => {
    // 取消正在进行的查询
    await utils.partnerManagement.getById.cancel({ id: newData.id });
    
    // 获取当前数据快照
    const previousData = utils.partnerManagement.getById.getData({ id: newData.id });
    
    // 乐观更新
    utils.partnerManagement.getById.setData({ id: newData.id }, (old) => ({
      ...old,
      ...newData
    }));
    
    return { previousData };
  },
  onError: (err, newData, context) => {
    // 出错时回滚
    utils.partnerManagement.getById.setData(
      { id: newData.id },
      context.previousData
    );
  },
  onSettled: () => {
    // 最终刷新数据
    utils.partnerManagement.getById.invalidate();
  }
});
```
