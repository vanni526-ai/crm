## 项目概述

**体验馆项目**（course_crm）是一套完整的课程交付管理系统，专为体验馆业务设计。系统涵盖客户管理、课程预约、订单管理、财务对账、合伙人分红等核心业务流程，支持多城市、多角色协作，是体验馆业务运营的数字化中枢。

### 核心价值

系统通过数字化手段解决了体验馆业务中的关键痛点：订单信息分散、财务对账繁琐、合伙人分红计算复杂、多城市协作困难。通过统一的数据管理和自动化计算，大幅提升了运营效率，降低了人工错误率。

### 业务模式

体验馆采用**多城市合伙人制**运营模式。总部负责课程研发和品牌管理，各城市合伙人负责本地市场开拓和课程交付。销售人员可跨城市接单，订单收入按预设比例分配给城市合伙人。系统自动计算每笔订单的分成金额，生成月度财务报表，确保分账透明准确。

## 系统架构

### 技术栈

系统采用现代化的全栈技术架构，前后端一体化开发，确保类型安全和开发效率。

**前端技术**：
- **React 19**：最新版本的React框架，提供高性能的用户界面
- **Tailwind CSS 4**：实用优先的CSS框架，快速构建响应式界面
- **shadcn/ui**：高质量的UI组件库，基于Radix UI
- **Wouter**：轻量级路由库，替代React Router
- **React Query**：强大的数据获取和缓存库

**后端技术**：
- **Node.js + Express**：成熟稳定的服务器框架
- **tRPC 11**：端到端类型安全的API框架，前后端共享类型定义
- **Drizzle ORM**：现代化的TypeScript ORM，类型安全的数据库操作
- **MySQL/TiDB**：关系型数据库，存储业务数据

**开发工具**：
- **TypeScript 5.9**：类型安全的JavaScript超集
- **Vite 7**：快速的前端构建工具
- **Vitest**：基于Vite的单元测试框架
- **pnpm**：高效的包管理工具

### 架构设计

系统采用**单体全栈架构**，前后端代码在同一仓库中，通过tRPC实现类型安全的API调用。这种架构简化了开发流程，避免了前后端类型不一致的问题，特别适合中小型团队快速迭代。

## 核心功能模块

### 1. 客户管理

客户管理模块是系统的基础模块，记录所有潜在客户和成交客户的信息。销售人员可以创建客户档案，记录沟通历史，跟进销售进度。系统支持按城市、销售人员、跟进状态等维度筛选客户，方便销售团队管理客户资源。

**主要功能**：
- 客户信息录入（姓名、电话、微信、城市、来源渠道）
- 客户跟进记录（沟通时间、内容、下次跟进计划）
- 客户状态管理（潜在客户、意向客户、成交客户、流失客户）
- 客户分配与转移（销售人员之间的客户转移）
- 客户搜索与筛选（按城市、销售人员、状态等条件）

**业务场景**：
- 销售人员通过线上线下渠道获取客户信息后，在系统中创建客户档案
- 定期跟进客户，记录沟通内容和客户反馈
- 客户成交后，系统自动关联订单信息
- 销售主管可以查看团队的客户跟进情况，及时发现问题

### 2. 课程预约

课程预约模块支持客户在线预约体验课程。系统展示可预约的课程列表，包括课程时间、地点、老师、剩余名额等信息。客户选择课程后提交预约，销售人员审核通过后生成正式订单。

**主要功能**：
- 课程日历展示（按月份、城市、老师筛选）
- 课程详情查看（时间、地点、老师、价格、名额）
- 在线预约提交（客户信息、预约课程、备注）
- 预约审核（销售人员确认预约，生成订单）
- 预约取消与改期（客户或销售人员发起）

**业务场景**：
- 客户通过微信公众号或小程序查看课程安排
- 选择合适的时间和地点，提交预约申请
- 销售人员收到预约通知，联系客户确认
- 审核通过后，系统自动生成订单并发送确认通知

### 3. 订单管理

订单管理模块是系统的核心，记录所有课程订单的完整生命周期。从订单创建、支付确认、课程交付到订单完成，每个环节都有详细记录。系统支持批量导入订单（从Excel或ICS日历文件），大幅提升订单录入效率。

**主要功能**：
- 订单创建（手动创建或预约转订单）
- 订单信息管理（客户、课程、老师、时间、地点、金额）
- 支付状态跟踪（未支付、部分支付、已支付、退款）
- 交付状态跟踪（待上课、已上课、已完成、已取消）
- 订单批量导入（Excel表格、ICS日历文件）
- 订单搜索与筛选（按城市、老师、日期、状态等）

**业务场景**：
- 销售人员确认客户预约后，创建正式订单
- 客户支付后，销售人员更新订单支付状态
- 课程结束后，系统记录实际上课情况（是否到课、课程反馈）
- 财务人员定期导出订单数据，进行对账和分成计算
- 运营人员批量导入Google Calendar的课程安排，自动生成订单

### 4. 财务管理

财务管理模块自动计算订单的收入分配，生成财务报表。系统根据预设的分成规则，计算每笔订单中城市合伙人、老师、销售人员的应得金额。支持按月度、城市、老师等维度生成财务报表，导出Excel文件供财务人员核对。

**主要功能**：
- 收入分成规则配置（城市合伙人比例、老师费用、车费）
- 订单分成自动计算（根据订单金额和分成规则）
- 月度财务报表生成（按城市、老师、销售人员统计）
- 合伙人分红明细（每个城市合伙人的应得金额）
- 老师费用统计（每位老师的月度收入）
- 销售业绩统计（每位销售人员的订单数量和金额）

**业务场景**：
- 财务人员每月初生成上月财务报表
- 系统自动计算每个城市合伙人的分红金额
- 导出Excel报表，发送给各城市合伙人核对
- 核对无误后，财务人员安排打款
- 老师和销售人员可以查看自己的收入明细

### 5. 合伙人管理

合伙人管理模块记录各城市合伙人的基本信息和分成配置。系统支持为每个城市设置不同的分成比例，灵活适应不同城市的运营模式。合伙人可以查看自己的分红明细和历史记录。

**主要功能**：
- 合伙人信息管理（姓名、城市、联系方式、合作协议）
- 分成比例配置（按城市设置分成比例）
- 分红明细查询（按月份查看分红金额和订单明细）
- 分红历史记录（历史分红金额、打款时间、打款凭证）
- 合伙人业绩统计（订单数量、收入金额、增长趋势）

**业务场景**：
- 新城市合伙人加入时，管理员创建合伙人档案并配置分成比例
- 合伙人每月查看自己的分红明细，核对订单数据
- 财务人员打款后，上传打款凭证到系统
- 管理员定期查看各城市合伙人的业绩，评估合作效果

### 6. 用户与权限管理

用户与权限管理模块支持多角色协作。系统预设了管理员、销售、财务、老师、客户等角色，每个角色有不同的功能权限。管理员可以创建用户账号，分配角色和权限，确保数据安全。

**主要功能**：
- 用户账号管理（创建、编辑、禁用、删除）
- 角色权限配置（管理员、销售、财务、老师、客户）
- 城市权限控制（用户只能查看和操作指定城市的数据）
- 登录认证（用户名密码登录、OAuth登录）
- 操作日志记录（记录用户的关键操作，便于审计）

**业务场景**：
- 新员工入职时，管理员创建账号并分配角色
- 销售人员只能查看和编辑自己负责的客户和订单
- 财务人员可以查看所有订单数据，但不能修改
- 城市合伙人只能查看本城市的数据
- 管理员可以查看所有用户的操作日志，发现异常行为

## 数据模型

系统的数据模型设计遵循关系型数据库的范式原则，确保数据一致性和查询效率。核心表包括用户表、客户表、订单表、课程表、老师表、合伙人表等。

### 核心数据表

**用户表 (users)**：
- 存储系统用户的基本信息和权限
- 字段：用户ID、用户名、密码、角色、城市权限、状态

**客户表 (customers)**：
- 存储客户的基本信息和跟进记录
- 字段：客户ID、姓名、电话、微信、城市、来源渠道、销售人员、状态

**订单表 (orders)**：
- 存储课程订单的完整信息
- 字段：订单ID、客户ID、课程ID、老师ID、上课时间、上课地点、订单金额、支付状态、交付状态、销售人员、城市

**课程表 (courses)**：
- 存储课程的基本信息
- 字段：课程ID、课程名称、课程类型、课程时长、课程价格、课程描述

**老师表 (teachers)**：
- 存储老师的基本信息和费用标准
- 字段：老师ID、姓名、电话、擅长课程、费用标准、所在城市

**合伙人表 (partners)**：
- 存储城市合伙人的信息和分成配置
- 字段：合伙人ID、姓名、城市、分成比例、联系方式、合作协议

### 数据关系

系统中的数据表通过外键关联，形成完整的业务数据网络。订单表是核心，关联了客户、课程、老师、销售人员、城市等多个维度的数据。通过这些关联关系，系统可以快速查询和统计各种业务数据。

```
客户表 ──┐
         ├──► 订单表 ◄──┬── 课程表
销售人员 ┘              ├── 老师表
                        ├── 城市表
                        └── 合伙人表
```

### 项目启动

克隆项目代码后，按以下步骤启动开发环境：

**1. 安装依赖**：
```bash
cd course_crm
pnpm install
```

**2. 配置环境变量**：

系统使用环境变量管理配置。开发环境的环境变量已由Manus平台自动注入，无需手动配置。如果需要自定义配置，可以创建`.env`文件：

# JWT密钥

# OAuth配置（如使用Manus OAuth）
OAUTH_SERVER_URL=https://api.manus.im
VITE_OAUTH_PORTAL_URL=https://auth.manus.im

**3. 初始化数据库**：
```bash

```

这个命令会自动创建数据库表结构。如果数据库中已有数据，命令会提示是否覆盖。

**4. 启动开发服务器**：
```bash

```

开发服务器启动后，访问 `http://localhost:3000` 即可看到系统界面。

**5. 启动Nginx反向代理（可选）**：

如果需要测试跨沙盒API访问，可以启动Nginx：

```bash
sudo systemctl start nginx
```

Nginx配置文件位于 `nginx.conf`，已配置好反向代理规则。

### 开发流程

系统采用**功能模块化**的开发方式，每个功能模块包含前端页面、后端API、数据库操作三部分。开发新功能时，按以下流程进行：

**1. 更新数据库表结构**：

在 `drizzle/schema.ts` 中定义新的数据表或修改现有表结构：

```typescript
// 示例：添加新表
export const newTable = sqliteTable('new_table', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  name: text('name').notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
});
```

修改后运行 `pnpm db:push` 更新数据库。

**2. 添加数据库查询函数**：

在 `server/db.ts` 中添加数据库查询函数：

```typescript
// 示例：查询函数
export async function getNewTableList() {
  return db.select().from(newTable).all();
}

export async function createNewTableItem(data: { name: string }) {
  return db.insert(newTable).values({
    name: data.name,
    createdAt: new Date(),
  }).run();
}
```

**3. 添加tRPC API接口**：

在 `server/routers.ts` 中添加API接口：

```typescript
// 示例：API接口
newTable: {
  list: publicProcedure.query(async () => {
    return await getNewTableList();
  }),
  
  create: protectedProcedure
    .input(z.object({ name: z.string() }))
    .mutation(async ({ input }) => {
      return await createNewTableItem(input);
    }),
},
```

**4. 开发前端页面**：

在 `client/src/pages/` 目录下创建新页面：

```typescript
// 示例：前端页面
import { trpc } from '@/lib/trpc';

export function NewTablePage() {
  const { data, isLoading } = trpc.newTable.list.useQuery();
  const createMutation = trpc.newTable.create.useMutation();

  if (isLoading) return <div>加载中...</div>;

  return (
    <div>
      <h1>新功能</h1>
      <ul>
        {data?.map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
      <button onClick={() => createMutation.mutate({ name: '测试' })}>
        添加
      </button>
    </div>
  );
}
```

**5. 注册路由**：

在 `client/src/App.tsx` 中注册新页面的路由：

```typescript
<Route path="/new-table" component={NewTablePage} />
```

**6. 编写测试**：

在 `server/` 目录下创建测试文件：

```typescript
// 示例：测试文件
import { describe, it, expect } from 'vitest';
import { getNewTableList } from './db';

describe('newTable', () => {
  it('should return list', async () => {
    const list = await getNewTableList();
    expect(Array.isArray(list)).toBe(true);
  });
});
```

运行测试：
```bash
pnpm test
```

**7. 提交代码**：

功能开发完成并通过测试后，提交代码：

```bash
git add .
git commit -m "feat: 添加新功能"
git push
```

### 生产环境部署

```bash

```

2. 将构建产物上传到服务器：
```bash
scp -r dist/ user@server:/path/to/app/
```

3. 在服务器上安装依赖并启动：
```bash
cd /path/to/app
pnpm install --prod
pnpm start
```

4. 配置Nginx反向代理和SSL证书：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

5. 配置进程管理器（如PM2）：
```bash
pm2 start dist/index.js --name course-crm
pm2 save
pm2 startup
```

## 常见问题

### 1. 如何添加新的用户角色？

在 `drizzle/schema.ts` 中修改用户表的角色枚举：

```typescript
export const users = sqliteTable('users', {
  // ...
  role: text('role', { enum: ['admin', 'sales', 'finance', 'teacher', 'customer', 'new_role'] }).notNull(),
});
```

然后在 `server/routers.ts` 中添加对应的权限检查逻辑。

### 2. 如何修改分成比例？

分成比例配置在合伙人表中。管理员可以在"合伙人管理"页面修改每个城市的分成比例。修改后，新订单会按新比例计算，历史订单不受影响。

### 3. 如何批量导入订单？

系统支持两种批量导入方式：

**方式1：Excel表格导入**
1. 下载订单导入模板
2. 按模板格式填写订单数据
3. 在"订单管理"页面点击"批量导入"
4. 上传Excel文件，系统自动解析并创建订单

**方式2：ICS日历文件导入**
1. 从Google Calendar导出ICS文件
2. 在"订单管理"页面点击"导入日历"
3. 上传ICS文件，系统自动解析课程事件并创建订单

### 4. 前端App如何连接后端API？

前端App需要使用完整的公网URL连接后端API，不能使用`localhost`。正确的配置方式：

```typescript
// ✅ 正确：使用公网URL
const API_BASE_URL = 'https://80-irlkkknuolzcky4z8bb9y-38095cbd.sg1.manus.computer/api';

// ❌ 错误：不能使用localhost
const API_BASE_URL = 'http://localhost:80';
```

详细配置方法请参考 `docs/frontend-app-api-config-guide.md`。

### 5. 如何配置CORS？

CORS配置在 `server/_core/index.ts` 中。系统已配置允许所有 `*.manus.computer` 域名访问。如果需要添加其他域名，修改CORS配置：

```typescript
const corsOptions = {
  origin: (origin, callback) => {
    // 添加允许的域名
    const allowedOrigins = [
      /\.manus\.computer$/,
      'https://your-domain.com',
    ];
    // ...
  },
};
```

### 6. 数据库迁移失败怎么办？

如果 `pnpm db:push` 失败，可能是数据库表结构冲突。解决方法：

1. 备份数据库：
```bash
mysqldump -u user -p course_crm > backup.sql
```

2. 删除冲突的表：
```sql
DROP TABLE IF EXISTS table_name;
```

3. 重新运行迁移：
```bash
pnpm db:push
```

4. 如果需要恢复数据：
```bash
mysql -u user -p course_crm < backup.sql
```
