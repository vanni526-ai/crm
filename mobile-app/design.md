# 课程预约App 界面设计文档

## 设计原则

本App严格遵循Apple Human Interface Guidelines (HIG)，以iOS原生应用为标准，确保在移动端竖屏（9:16）和单手操作场景下的最佳用户体验。

## 屏幕列表

| 屏幕名称 | 路由 | 描述 |
|---------|------|------|
| 登录页 | /login | 用户登录入口 |
| 首页/约课 | /(tabs)/index | 约课流程主入口 |
| 我的预约 | /(tabs)/my-bookings | 查看预约记录 |
| 个人中心 | /(tabs)/profile | 个人信息与设置 |
| 预约确认 | /booking/confirm | 确认预约信息 |

## 约课流程设计

### 流程步骤
```
选择城市 → 选择时间 → 选择老师 → 选择课程 → 确认预约
```

### 城市选择
- 展示形式：网格卡片（2列）
- 数据来源：后端 API cityPartnerConfig.list
- 显示内容：城市名称（带拼音首字母）
- 交互：单选，选中后高亮显示

### 时间选择
- 日期选择：水平滚动日期选择器，显示未来7天
- 时间段选择：网格按钮（每整点1小时为1节课）
- 可选时间：9:00-21:00（共12个时段）
- 交互：日期单选 + 时间段单选

### 老师选择
- 展示形式：横向滚动卡片
- 卡片内容：老师头像、姓名、属性标签（S/M/SW）、介绍
- 数据来源：后端 API teachers.list
- 交互：单选，选中卡片边框高亮

### 课程选择
- 展示形式：瀑布流（2列）
- 卡片内容：课程名称、价格、时长、程度标签、描述
- 数据来源：后端 API courses.list
- 筛选功能：按价格、程度、时长筛选排序
- 交互：点击预约按钮进入确认页

## 关键用户流程

### 约课流程
1. 用户打开App → 首页显示约课入口
2. 点击"开始约课" → 进入城市选择
3. 选择城市 → 自动进入时间选择
4. 选择日期和时间段 → 自动进入老师选择
5. 选择老师 → 自动进入课程选择
6. 选择课程点击"预约" → 进入确认页
7. 确认信息 → 提交预约

### 查看预约
1. 点击底部Tab"我的预约" → 显示预约列表
2. 点击预约卡片 → 查看预约详情
