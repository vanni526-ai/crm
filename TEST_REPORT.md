## 🔍 发现的问题

### 🔴 严重问题

#### 1. customerId字段类型限制导致测试失败
**问题描述**:  
- 数据库schema中`customerId`字段使用`int()`类型
- MySQL INT最大值: 2,147,483,647
- 测试用例使用时间戳作为customerId(如1,769,250,341,915),超出INT范围
- 导致所有涉及订单创建的测试失败

**影响范围**:
- `server/churnRisk.test.ts` - 5个测试全部失败
- `server/orderOperations.test.ts` - 3个测试失败
- `server/orderStatus.test.ts` - 可能受影响

**错误信息**:
```
Error: Out of range value for column 'customerId' at row 1
```

**解决方案**:
1. **短期**: 修改测试用例,使用小于INT最大值的ID(如1-1000000)
2. **长期**: 修改schema,将`customerId`改为`bigint()`类型,支持更大范围的ID

**优先级**: 🔴 高 - 影响测试和可能的实际业务
