## 测试内容
验证课程排课页面是否正确显示交付城市和交付教室信息

## 测试结果

### ✅ 全部列表视图(表格)

  - "交付教室"列

- 当订单匹配成功时,显示对应的城市和教室名称
- 当订单未匹配时,显示"-"

## 数据来源说明

课程排课页面的城市和教室信息来自关联的订单数据:

1. **数据流程**:
   - 课程排课表(`schedules`) → 通过`channelOrderNo`字段关联 → 订单表(`orders`)
   - 订单表包含`deliveryCity`(交付城市)和`deliveryRoom`(交付教室)字段
   - 前端通过`trpc.schedules.listWithOrderInfo.useQuery()`获取包含订单信息的排课数据

2. **显示逻辑**:
   ```typescript
   {schedule.matchedOrder?.deliveryCity && (
     <div>城市: {schedule.matchedOrder.deliveryCity}</div>
   )}
   {schedule.matchedOrder?.deliveryRoom && (
     <div>教室: {schedule.matchedOrder.deliveryRoom}</div>
   )}
   ```

3. **教室匹配状态**:
   - 已匹配教室ID的订单: 85个
   - 未匹配教室ID的订单: 136个
   - 未匹配原因:
     - 城市不存在(如"互联网课"、"北京"、"广州"等)
     - 教室名称不匹配(如"捷运大厦16D"、"天津教室"等)

## 建议优化

1. **完善城市数据**: 将"北京"、"广州"、"成都"、"杭州"等常见城市添加到cities表
2. **教室名称标准化**: 统一教室命名规则,避免"404教室"、"404"、"捷运大厦16D"等多种写法
3. **自动匹配优化**: 增强模糊匹配算法,提高教室自动匹配成功率
4. **数据清洗**: 对历史订单数据进行清洗,统一城市和教室名称格式
