# 老师接口文档

## 概述

本文档提供课程交付CRM系统中老师相关的完整API接口说明，供前端APP开发团队使用。所有接口基于tRPC协议，使用类型安全的请求和响应。

---

## 接口基础信息

**接口协议**: tRPC  
**接口前缀**: `/api/trpc/teachers`  
**认证方式**: Session Cookie（部分接口需要登录，部分接口公开访问）

---

## 1. 获取老师列表

### 接口路径
```
teachers.list
```

### 请求方式
`query`（查询类型，无需请求参数）

### 权限要求
公开接口（`publicProcedure`），无需登录

### 请求参数
无

### 响应字段说明

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `id` | `number` | 老师用户ID（主键） | `15362121` |
| `name` | `string` | 老师姓名 | `"琴酒"` |
| `nickname` | `string \| null` | 老师昵称 | `"酒酒"` |
| `phone` | `string \| null` | 手机号码 | `"13800138000"` |
| `customerType` | `string \| null` | 客户类型 | `"VIP"` |
| `isActive` | `number` | 激活状态（1=激活，0=未激活） | `1` |
| `avatarUrl` | `string \| null` | 头像URL | `"https://files.manuscdn.com/..."` |
| `teacherAttribute` | `"S" \| "M" \| "Switch" \| null` | 老师属性 | `"S"` |
| `status` | `string \| null` | 老师状态 | `"活跃"` |
| `notes` | `string \| null` | 老师备注（详细技能描述） | `"丝zu、sp、踩ta、刑fa、羞ru、k9、tk、角色/情景扮演、捆绑、电击"` |
| `cities` | `string \| null` | 关联城市（分号分隔） | `"重庆;武汉"` |

### 字段详细说明

#### `isActive` - 激活状态
- **类型**: `number`
- **取值**: `1`（激活）或 `0`（未激活）
- **说明**: 表示老师账号是否处于激活状态，只有激活的老师才能接单和排课

#### `teacherAttribute` - 老师属性
- **类型**: `"S" | "M" | "Switch" | null`
- **取值说明**:
  - `"S"`: S属性老师
  - `"M"`: M属性老师
  - `"Switch"`: Switch属性老师（可S可M）
  - `null`: 未设置属性

#### `status` - 老师状态
- **类型**: `string | null`
- **常见取值**: `"活跃"`, `"休假"`, `"离职"`, `"待定"`
- **说明**: 表示老师当前的工作状态

#### `notes` - 老师备注
- **类型**: `string | null`
- **说明**: 存储老师的详细技能描述，包括擅长的项目（如k8、k9、sp、踩踏、刑罚、羞辱、角色扮演等）
- **注意**: 此字段可能包含敏感内容，前端展示时需要注意隐私保护

#### `cities` - 关联城市
- **类型**: `string | null`
- **格式**: 多个城市用分号（`;`）分隔
- **示例**: `"重庆;武汉"`, `"北京"`
- **说明**: 表示老师可以在哪些城市提供服务
- **解析方式**: 
  ```javascript
  const cityArray = cities ? cities.split(';').map(c => c.trim()) : [];
  ```

## 数据说明

### 老师激活状态（isActive）

系统中所有老师的激活状态默认为 `1`（激活），因为：
1. users表没有独立的isActive字段
2. 通过`deletedAt`字段判断老师是否被删除（软删除）
3. 只有未删除的老师会出现在列表中

### 老师与城市的关联关系

老师与城市的关联关系存储在`userRoleCities`表中：
- 一个老师可以关联多个城市
- 关联关系通过`userId`和`role='teacher'`查询
- 前端展示时，多个城市用分号分隔

### 老师备注字段（notes）

老师备注统一存储在`teachers.notes`字段中：
- 对于主角色为teacher的用户，数据存储在teachers表
- 对于多角色用户（如主角色为user但包含teacher角色），数据也存储在teachers表
- 系统确保所有有teacher角色的用户在teachers表都有对应记录

## 前端调用示例

### React + tRPC Hooks

```typescript
import { trpc } from '@/lib/trpc';

// 1. 获取老师列表
function TeacherList() {
  const { data: teachers, isLoading } = trpc.teachers.list.useQuery();
  
  if (isLoading) return <div>加载中...</div>;
  
  return (
    <div>
      {teachers?.map(teacher => (
        <div key={teacher.id}>
          <img src={teacher.avatarUrl || DEFAULT_AVATAR} alt={teacher.name} />
          <h3>{teacher.name}</h3>
          <p>属性: {teacher.teacherAttribute || '未设置'}</p>
          <p>状态: {teacher.status || '未知'}</p>
          <p>城市: {teacher.cities || '未设置'}</p>
          <p>备注: {teacher.notes || '无'}</p>
        </div>
      ))}
    </div>
  );
}

// 2. 获取单个老师详情
function TeacherDetail({ teacherId }: { teacherId: number }) {
  const { data: teacher } = trpc.teachers.getById.useQuery({ id: teacherId });
  
  if (!teacher) return <div>老师不存在</div>;
  
  return (
    <div>
      <h2>{teacher.name}</h2>
      <p>激活状态: {teacher.isActive === 1 ? '已激活' : '未激活'}</p>
      <p>关联城市: {teacher.cities?.split(';').join(', ') || '未设置'}</p>
    </div>
  );
}

// 3. 获取老师统计数据
function TeacherStats({ teacherId }: { teacherId: number }) {
  const { data: stats } = trpc.teachers.getStats.useQuery({
    teacherId,
    startDate: new Date('2026-01-01'),
    endDate: new Date('2026-02-28'),
  });
  
  return (
    <div>
      <p>总订单数: {stats?.totalOrders}</p>
      <p>总收入: ¥{stats?.totalRevenue}</p>
      <p>总课时: {stats?.totalHours}小时</p>
      <p>平均评分: {stats?.averageRating}分</p>
    </div>
  );
}
```

## 注意事项

1. **敏感信息保护**: `notes`字段可能包含敏感内容，前端展示时需要注意隐私保护
2. **城市字段解析**: `cities`字段是分号分隔的字符串，需要前端自行解析为数组
3. **默认头像**: 当`avatarUrl`为`null`时，使用默认头像URL: `https://files.manuscdn.com/user_upload_by_module/session_file/310519663214896586/JopHWzeEmqAYxCyT.png`
4. **激活状态**: 所有列表中的老师`isActive`字段均为`1`，因为已删除的老师不会出现在列表中
5. **类型安全**: 使用tRPC客户端调用接口时，TypeScript会自动提供类型检查和智能提示
