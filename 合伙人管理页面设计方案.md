# 合伙人管理页面设计方案

## 一、需求分析

根据提供的Excel大纲，合伙人管理功能需要包含以下模块：

### 1. 基本信息
- 账号（关联users表）
- 手机号
- 角色：城市合伙人

### 2. 合同信息
- 分红比例（根据合同内容）
- 分红规则
- 品牌加盟费
- 技术服务赕
- 后付款总金额
- 后付款扣款规则
- 合同更新（记录合同修订历史）
- 合同起止日期

### 3. 城市管理
- 可管理多个城市
- 每个城市独立统计

### 4. 订单统计
- 销售额
- 订单数量
- 按城市分组统计

### 5. 费用明细
- 房租
- 物业
- 水电
- 耗材
- 老师费用
- 车费
- 其他费用
- 总费用
- 合同后付款
- 合同后付款总金额
- 合同后付款未结清余额
- 月度分红明细单（按月份生成PDF报表）
- 合同后付款
- 合同后付款总金额
- 合同后付款未结清余额

### 6. 分红计算
- 分红金额（合伙人费）
- 基于订单自动计算

### 7. 收款账户
- 开户名
- 开户行
- 账号
- 历史分红流水（记录每次分红转账的时间、金额、备注）

## 四、数据库设计

### 新增表：partners（合伙人表）

```sql
CREATE TABLE partners (
  id INT AUTO_INCREMENT PRIMARY KEY,
  userId INT NOT NULL COMMENT '关联users表',
  contractStartDate DATE COMMENT '合同开始日期',
  contractEndDate DATE COMMENT '合同结束日期',
  profitSharingRules TEXT COMMENT '分红规则',
  brandFee DECIMAL(10,2) DEFAULT 0 COMMENT '品牌加盟费',
  techServiceFee DECIMAL(10,2) DEFAULT 0 COMMENT '技术服务费',
  deferredPaymentTotal DECIMAL(10,2) DEFAULT 0 COMMENT '后付款总金额',
  deferredPaymentPaid DECIMAL(10,2) DEFAULT 0 COMMENT '已支付后付款',
  deferredPaymentRules TEXT COMMENT '后付款扣款规则',
  bankAccountName VARCHAR(100) COMMENT '开户名',
  bankName VARCHAR(200) COMMENT '开户行',
  bankAccountNo VARCHAR(50) COMMENT '银行账号',
  createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (userId) REFERENCES users(id)
);
```

### 新增表：partner_expenses（合伙人费用明细表）

```sql
CREATE TABLE partner_expenses (
  id INT AUTO_INCREMENT PRIMARY KEY,
  partnerId INT NOT NULL COMMENT '关联partners表',
  city VARCHAR(50) NOT NULL COMMENT '城市',
  expenseMonth DATE NOT NULL COMMENT '费用月份',
  rentFee DECIMAL(10,2) DEFAULT 0 COMMENT '房租',
  propertyFee DECIMAL(10,2) DEFAULT 0 COMMENT '物业费',
  utilityFee DECIMAL(10,2) DEFAULT 0 COMMENT '水电费',
  consumablesFee DECIMAL(10,2) DEFAULT 0 COMMENT '耗材费',
  teacherFee DECIMAL(10,2) DEFAULT 0 COMMENT '老师费用',
  transportFee DECIMAL(10,2) DEFAULT 0 COMMENT '车费',
  otherFee DECIMAL(10,2) DEFAULT 0 COMMENT '其他费用',
  totalFee DECIMAL(10,2) DEFAULT 0 COMMENT '总费用',
  notes TEXT COMMENT '备注',
  createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (partnerId) REFERENCES partners(id),
  INDEX idx_partner_month (partnerId, expenseMonth)
);
```

## 九、注意事项

1. **权限控制**：只有管理员和财务角色可以访问合伙人管理功能
2. **数据一致性**：合伙人的城市必须与`user_role_cities`表保持同步
3. **分红计算**：自动从订单表聚合计算，使用已有的`calculatePartnerFee`函数
4. **后付款扣款**：需要定时任务或手动触发扣款操作
5. **敏感信息**：银行账号等敏感信息需要加密存储或脱敏显示
