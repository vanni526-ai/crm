# 订单交付状态功能 - 前端接口文档

## 概述

订单新增 `deliveryStatus`（交付状态）字段，支持两种状态：
- `undelivered` - 未交付（默认值）
- `delivered` - 已交付

---

## 一、数据库字段

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `deliveryStatus` | ENUM('undelivered', 'delivered') | `undelivered` | 订单交付状态 |

该字段已添加到 `orders` 表中，所有新建订单默认为 `undelivered`。

---

## 二、API 接口

### 2.1 查询订单列表（已包含交付状态）

**接口**: `orders.myOrders`

**请求参数**:
```typescript
{
  status?: 'all' | 'pending' | 'paid' | 'completed' | 'cancelled' | 'refunded';
  limit?: number;
  offset?: number;
}
```

**响应数据**（每个订单对象新增字段）:
```typescript
{
  orders: [
    {
      id: number;
      orderNo: string;
      customerName: string;
      paymentAmount: string;
      courseAmount: string;
      deliveryCity: string;
      deliveryCourse: string;
      status: string;
      deliveryStatus: 'undelivered' | 'delivered';  // 新增字段
      createdAt: string;
      // ... 其他字段
    }
  ],
  total: number;
}
```

### 2.2 更新单个订单交付状态

**接口**: `orders.updateDeliveryStatus`

**权限**: 需要销售或管理员权限（salesOrAdmin）

**请求参数**:
```typescript
{
  id: number;                                        // 订单ID
  deliveryStatus: 'undelivered' | 'delivered';       // 目标交付状态
}
```

**响应数据**:
```typescript
{
  success: boolean;
}
```

### 2.3 批量更新订单交付状态

**接口**: `orders.batchUpdateDeliveryStatus`

**权限**: 需要销售或管理员权限（salesOrAdmin）

**请求参数**:
```typescript
{
  ids: number[];                                     // 订单ID数组
  deliveryStatus: 'undelivered' | 'delivered';       // 目标交付状态
}
```

**响应数据**:
```typescript
{
  success: boolean;
  count: number;    // 更新的订单数量
}
```

### 2.4 创建订单时指定交付状态

**接口**: `orders.userCreate` / `orders.create`

在创建订单时可选传入 `deliveryStatus`，不传则默认为 `undelivered`。

## 三、SDK 使用示例

### 3.1 安装与初始化

```typescript
import { createApiClient } from './api-client';
const api = createApiClient();
```

### 3.2 查询订单列表（包含交付状态）

```typescript
const result = await api.orders.myOrders({ limit: 20, offset: 0 });
result.orders.forEach(order => {
  console.log(`订单 ${order.orderNo}: ${order.deliveryStatus === 'delivered' ? '已交付' : '未交付'}`);
});
```

### 3.3 更新单个订单交付状态

```typescript
// 标记为已交付
await api.orders.updateDeliveryStatus(orderId, 'delivered');

// 标记为未交付
await api.orders.updateDeliveryStatus(orderId, 'undelivered');
```

## 四、React Native 集成示例

### 4.1 订单列表中显示交付状态

```tsx
import React from 'react';
import { View, Text, TouchableOpacity, FlatList, StyleSheet } from 'react-native';

const OrderItem = ({ order, onToggleDelivery }) => (
  <View style={styles.orderItem}>
    <View style={styles.orderInfo}>
      <Text style={styles.orderNo}>{order.orderNo}</Text>
      <Text style={styles.customerName}>{order.customerName}</Text>
      <Text style={styles.amount}>¥{order.paymentAmount}</Text>
    </View>
    <TouchableOpacity
      style={[
        styles.deliveryBadge,
        order.deliveryStatus === 'delivered' ? styles.delivered : styles.undelivered,
      ]}
      onPress={() => onToggleDelivery(order)}
    >
      <Text style={[
        styles.deliveryText,
        order.deliveryStatus === 'delivered' ? styles.deliveredText : styles.undeliveredText,
      ]}>
        {order.deliveryStatus === 'delivered' ? '已交付' : '未交付'}
      </Text>
    </TouchableOpacity>
  </View>
);

const OrderList = () => {
  const [orders, setOrders] = React.useState([]);
  const api = useApiClient(); // 自定义hook获取api实例

  const loadOrders = async () => {
    const result = await api.orders.myOrders({ limit: 50 });
    setOrders(result.orders);
  };

  const toggleDeliveryStatus = async (order) => {
    const newStatus = order.deliveryStatus === 'delivered' ? 'undelivered' : 'delivered';
    try {
      await api.orders.updateDeliveryStatus(order.id, newStatus);
      // 乐观更新
      setOrders(prev =>
        prev.map(o => o.id === order.id ? { ...o, deliveryStatus: newStatus } : o)
      );
    } catch (error) {
      Alert.alert('错误', '更新交付状态失败');
    }
  };

  React.useEffect(() => { loadOrders(); }, []);

  return (
    <FlatList
      data={orders}
      keyExtractor={item => String(item.id)}
      renderItem={({ item }) => (
        <OrderItem order={item} onToggleDelivery={toggleDeliveryStatus} />
      )}
    />
  );
};

const styles = StyleSheet.create({
  orderItem: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    padding: 16,
    borderBottomWidth: 1,
    borderBottomColor: '#eee',
  },
  orderInfo: { flex: 1 },
  orderNo: { fontSize: 14, fontWeight: '600' },
  customerName: { fontSize: 13, color: '#666', marginTop: 4 },
  amount: { fontSize: 15, fontWeight: '700', color: '#333', marginTop: 4 },
  deliveryBadge: {
    paddingHorizontal: 12,
    paddingVertical: 6,
    borderRadius: 16,
    borderWidth: 1,
  },
  delivered: {
    backgroundColor: '#16a34a',
    borderColor: '#16a34a',
  },
  undelivered: {
    backgroundColor: 'transparent',
    borderColor: '#d1d5db',
  },
  deliveryText: { fontSize: 12, fontWeight: '600' },
  deliveredText: { color: '#fff' },
  undeliveredText: { color: '#6b7280' },
});
```

### 4.2 筛选未交付/已交付订单

```tsx
const [filter, setFilter] = React.useState<'all' | 'undelivered' | 'delivered'>('all');

const filteredOrders = React.useMemo(() => {
  if (filter === 'all') return orders;
  return orders.filter(o => o.deliveryStatus === filter);
}, [orders, filter]);

// 筛选按钮组
<View style={{ flexDirection: 'row', gap: 8, padding: 12 }}>
  {['all', 'undelivered', 'delivered'].map(f => (
    <TouchableOpacity
      key={f}
      style={[styles.filterBtn, filter === f && styles.filterBtnActive]}
      onPress={() => setFilter(f)}
    >
      <Text>{f === 'all' ? '全部' : f === 'delivered' ? '已交付' : '未交付'}</Text>
    </TouchableOpacity>
  ))}
</View>
```

## 五、注意事项

1. **默认值**: 所有新创建的订单默认 `deliveryStatus = 'undelivered'`
2. **权限控制**: 更新交付状态需要销售或管理员权限
3. **历史数据**: 已有订单的 `deliveryStatus` 默认为 `undelivered`
4. **乐观更新**: 前端建议使用乐观更新策略，先更新UI再等待接口响应
5. **状态独立**: `deliveryStatus` 与 `status`（支付状态）是两个独立字段，互不影响
