## 一、功能概述

本文档为前端App开发团队提供"新用户注册"功能的完整后端接口说明。用户通过手机号和密码注册后，可使用相同凭证登录App。

### 1.1 业务流程

```
用户输入手机号 → 输入密码 → 确认密码 → 提交注册 → 注册成功 → 自动登录/跳转登录页
```

### 1.2 后端现状分析

| 组件 | 状态 | 说明 |
|------|------|------|
| 数据库表 | ✅ 已存在 | `users`表已包含`phone`和`password`字段 |
| 登录接口 | ✅ 已存在 | `auth.loginWithUserAccount`支持手机号登录 |
| 注册接口 | ❌ 需新增 | 需要创建`auth.register`接口 |

## 二、数据库结构

### 2.1 用户表(users)

后端已有完整的用户表结构，支持手机号注册和登录：

```sql
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  openId VARCHAR(64) NOT NULL UNIQUE,      -- 用户唯一标识(注册时自动生成)
  name TEXT,                                -- 用户名(可选)
  nickname VARCHAR(50),                     -- 昵称(可选)
  email VARCHAR(320),                       -- 邮箱(可选)
  phone VARCHAR(20),                        -- 手机号(用于登录)
  password VARCHAR(255),                    -- 加密后的密码(bcrypt)
  loginMethod VARCHAR(64),                  -- 登录方式标识
  role ENUM('admin','sales','finance','user') DEFAULT 'user', -- 角色
  isActive BOOLEAN DEFAULT TRUE,            -- 是否激活
  createdAt TIMESTAMP DEFAULT NOW(),        -- 创建时间
  updatedAt TIMESTAMP DEFAULT NOW() ON UPDATE NOW(), -- 更新时间
  lastSignedIn TIMESTAMP DEFAULT NOW()      -- 最后登录时间
);
```

### 2.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `phone` | VARCHAR(20) | ✅ | 手机号，用于登录凭证，建议唯一 |
| `password` | VARCHAR(255) | ✅ | 密码，使用bcrypt加密存储 |
| `openId` | VARCHAR(64) | 自动 | 系统自动生成的唯一标识 |
| `role` | ENUM | 自动 | 默认为`user`(普通用户) |
| `isActive` | BOOLEAN | 自动 | 默认为`true`(激活状态) |

## 三、API接口规范

### 3.1 注册接口(需后端新增)

**接口名称**: `auth.register`

**请求方式**: POST (tRPC mutation)

**接口路径**: `/api/trpc/auth.register`

#### 请求参数

```typescript
interface RegisterInput {
  phone: string;      // 手机号(必填,11位)
  password: string;   // 密码(必填,6-20位)
  name?: string;      // 用户名(可选)
  nickname?: string;  // 昵称(可选)
}
```

#### 响应格式

```typescript
interface RegisterResult {
  success: boolean;           // 是否成功
  message: string;            // 提示信息
  token?: string;             // JWT Token(注册成功后自动登录)
  user?: {
    id: number;               // 用户ID
    openId: string;           // 用户唯一标识
    phone: string;            // 手机号
    name: string;             // 用户名
    role: string;             // 角色
  };
}
```

#### 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| `CONFLICT` | 409 | 手机号已被注册 |
| `BAD_REQUEST` | 400 | 参数格式错误(手机号/密码不符合要求) |
| `INTERNAL_SERVER_ERROR` | 500 | 服务器内部错误 |

#### 请求示例

```bash
# 使用curl测试
curl -X POST "https://your-api-domain/api/trpc/auth.register" \
  -H "Content-Type: application/json" \
  -d '{"json":{"phone":"13800138000","password":"123456"}}'
```

#### 响应示例

**成功响应**:
```json
{
  "result": {
    "data": {
      "json": {
        "success": true,
        "message": "注册成功",
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "user": {
          "id": 123,
          "openId": "user_abc123xyz",
          "phone": "13800138000",
          "name": "",
          "role": "user"
        }
      }
    }
  }
}
```

**失败响应(手机号已存在)**:
```json
{
  "error": {
    "message": "该手机号已被注册",
    "code": -32603,
    "data": {
      "code": "CONFLICT",
      "httpStatus": 409
    }
  }
}
```

### 4.2 直接HTTP请求

如果不使用SDK，可以直接发送HTTP请求：

```typescript
// React Native示例
async function register(phone: string, password: string) {
  const response = await fetch(
    'https://your-api-domain/api/trpc/auth.register',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        json: { phone, password }
      }),
    }
  );
  
  const data = await response.json();
  
  if (data.result?.data?.json?.success) {
    // 保存Token
    const token = data.result.data.json.token;
    await AsyncStorage.setItem('crm_auth_token', token);
    return data.result.data.json;
  } else {
    throw new Error(data.error?.message || '注册失败');
  }
}
```

### 4.3 Token传递方式

由于Cloudflare CDN会过滤Authorization头，需要通过URL参数传递Token：

```typescript
// 在请求URL中附加Token
const token = await AsyncStorage.getItem('crm_auth_token');
const url = `https://your-api-domain/api/trpc/orders.myOrders?_token=${encodeURIComponent(token)}`;
```

SDK已自动处理此逻辑，无需手动操作。

## 五、前端UI建议

### 5.1 注册页面组件结构

```tsx
// RegisterScreen.tsx
import React, { useState } from 'react';
import { View, TextInput, Button, Alert, StyleSheet } from 'react-native';

export function RegisterScreen({ navigation }) {
  const [phone, setPhone] = useState('');
  const [password, setPassword] = useState('');
  const [confirmPassword, setConfirmPassword] = useState('');
  const [loading, setLoading] = useState(false);

  const validatePhone = (phone: string) => {
    return /^1[3-9]\d{9}$/.test(phone);
  };

  const handleRegister = async () => {
    // 1. 前端校验
    if (!validatePhone(phone)) {
      Alert.alert('提示', '请输入正确的手机号');
      return;
    }
    if (password.length < 6) {
      Alert.alert('提示', '密码至少6位');
      return;
    }
    if (password !== confirmPassword) {
      Alert.alert('提示', '两次密码输入不一致');
      return;
    }

    // 2. 调用注册接口
    setLoading(true);
    try {
      const result = await api.auth.register({ phone, password });
      if (result.success) {
        Alert.alert('成功', '注册成功', [
          { text: '确定', onPress: () => navigation.navigate('Home') }
        ]);
      }
    } catch (error) {
      Alert.alert('注册失败', error.message);
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="请输入手机号"
        keyboardType="phone-pad"
        maxLength={11}
        value={phone}
        onChangeText={setPhone}
      />
      <TextInput
        style={styles.input}
        placeholder="请输入密码(至少6位)"
        secureTextEntry
        value={password}
        onChangeText={setPassword}
      />
      <TextInput
        style={styles.input}
        placeholder="请确认密码"
        secureTextEntry
        value={confirmPassword}
        onChangeText={setConfirmPassword}
      />
      <Button
        title={loading ? '注册中...' : '注册'}
        onPress={handleRegister}
        disabled={loading}
      />
      <Button
        title="已有账号?去登录"
        onPress={() => navigation.navigate('Login')}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, justifyContent: 'center' },
  input: { 
    borderWidth: 1, 
    borderColor: '#ddd', 
    padding: 12, 
    marginBottom: 15, 
    borderRadius: 8 
  },
});
```

## 六、后端实现参考

以下是后端需要实现的注册接口参考代码：

```typescript
// server/authRouter.ts - 新增register接口

import { z } from "zod";
import { publicProcedure } from "./_core/trpc";
import { TRPCError } from "@trpc/server";
import { getDb } from "./db";
import { users } from "../drizzle/schema";
import { eq } from "drizzle-orm";
import { hashPassword } from "./passwordUtils";
import jwt from "jsonwebtoken";
import { v4 as uuidv4 } from "uuid";

const JWT_SECRET = process.env.JWT_SECRET || "your-secret-key";
const TOKEN_EXPIRY = "24h";

// 在authRouter中添加:
register: publicProcedure
  .input(
    z.object({
      phone: z.string()
        .min(11, "手机号格式错误")
        .max(11, "手机号格式错误")
        .regex(/^1[3-9]\d{9}$/, "请输入正确的手机号"),
      password: z.string()
        .min(6, "密码至少6位")
        .max(20, "密码最多20位"),
      name: z.string().optional(),
      nickname: z.string().optional(),
    })
  )
  .mutation(async ({ input, ctx }) => {
    const drizzle = await getDb();
    if (!drizzle) {
      throw new TRPCError({
        code: "INTERNAL_SERVER_ERROR",
        message: "数据库连接失败",
      });
    }

    // 1. 检查手机号是否已注册
    const existingUser = await drizzle
      .select()
      .from(users)
      .where(eq(users.phone, input.phone))
      .limit(1);

    if (existingUser.length > 0) {
      throw new TRPCError({
        code: "CONFLICT",
        message: "该手机号已被注册",
      });
    }

    // 2. 加密密码
    const hashedPassword = await hashPassword(input.password);

    // 3. 生成唯一标识
    const openId = `user_${uuidv4().replace(/-/g, '').substring(0, 16)}`;

    // 4. 创建用户
    const result = await drizzle.insert(users).values({
      openId,
      phone: input.phone,
      password: hashedPassword,
      name: input.name || input.phone,
      nickname: input.nickname,
      role: "user",
      isActive: true,
      loginMethod: "phone",
    });

    const userId = result[0].insertId;

    // 5. 生成JWT Token(注册后自动登录)
    const token = jwt.sign(
      {
        id: userId,
        openId,
        name: input.name || input.phone,
        role: "user",
      },
      JWT_SECRET,
      { expiresIn: TOKEN_EXPIRY }
    );

    // 6. 设置session cookie
    ctx.res?.cookie("session", token, {
      httpOnly: true,
      secure: process.env.NODE_ENV === "production",
      sameSite: "lax",
      maxAge: 7 * 24 * 60 * 60 * 1000,
    });

    return {
      success: true,
      message: "注册成功",
      token,
      user: {
        id: userId,
        openId,
        phone: input.phone,
        name: input.name || input.phone,
        role: "user",
      },
    };
  }),
```

## 八、安全建议

### 8.1 密码安全

后端使用bcrypt进行密码加密，前端无需额外处理。建议：

1. **密码强度提示**: 前端可添加密码强度指示器
2. **防暴力破解**: 后端已有登录失败次数限制(建议实现)
3. **HTTPS传输**: 所有请求必须通过HTTPS

### 8.2 Token安全

1. **安全存储**: 使用AsyncStorage存储Token，避免明文存储
2. **自动刷新**: 使用SDK的`autoRefreshIfNeeded()`方法
3. **退出清理**: 登出时清除本地Token

## 九、常见问题

### Q1: 注册接口返回"数据库连接失败"怎么办?
**A**: 这是后端服务问题，请联系后端开发人员检查数据库连接配置。

### Q2: Token过期后如何处理?
**A**: 使用SDK的`refreshToken()`方法刷新Token，过期7天内可刷新。超过7天需重新登录。

### Q3: 如何判断用户是否已登录?
**A**: 使用SDK的`isLoggedIn()`方法检查，或调用`auth.me`接口验证。

### Q4: 注册成功后需要跳转到登录页吗?
**A**: 不需要。注册接口会返回Token，前端保存后可直接进入首页。
