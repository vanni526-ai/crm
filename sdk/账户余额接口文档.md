# 账户余额接口文档

> 本文档面向前端App开发团队，描述账户余额查询、充值、消费扣除、退款相关的后端API接口。
> 所有接口已通过后端16个单元测试验证，接口稳定可用。

---

## 一、数据库结构

### 1.1 余额字段

余额存储在 `customers`（业务客户）表中：

| 字段 | 类型 | 说明 |
|------|------|------|
| `accountBalance` | decimal(10,2) | 账户余额，默认"0.00" |

> **重要**：余额绑定在业务客户（customers）上，而非用户账号（users）。App用户注册/登录时会自动创建关联的业务客户记录。

### 1.2 流水表 `accountTransactions`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 流水ID，自增主键 |
| `customerId` | int | 关联的业务客户ID |
| `type` | enum | 类型：`recharge`(充值)、`consume`(消费)、`refund`(退款) |
| `amount` | decimal(10,2) | 金额 |
| `balanceBefore` | decimal(10,2) | 操作前余额 |
| `balanceAfter` | decimal(10,2) | 操作后余额 |
| `orderId` | int/null | 关联订单ID（消费/退款时有值） |
| `orderNo` | varchar/null | 关联订单号 |
| `notes` | text/null | 备注 |
| `operatorId` | int/null | 操作人ID |
| `operatorName` | varchar/null | 操作人姓名 |
| `createdAt` | datetime | 创建时间 |

## 二、接口列表

### 2.1 查询当前用户余额

**接口路径**：`account.getMyBalance`（tRPC query）

**认证要求**：需要登录（Token）

**请求参数**：无

**响应格式**：

```json
{
  "success": true,
  "data": {
    "balance": "800.00",
    "customerId": 480003,
    "customerName": "张三"
  }
}
```

**特殊情况**：用户未关联业务客户时返回：

```json
{
  "success": true,
  "data": {
    "balance": "0.00",
    "customerId": null,
    "customerName": null
  }
}
```

**SDK调用**：

```typescript
const result = await api.account.getMyBalance();
if (result.success) {
  console.log('余额:', result.data.balance);
}
```

---

### 2.2 查询当前用户流水

**接口路径**：`account.getMyTransactions`（tRPC query）

**认证要求**：需要登录（Token）

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `limit` | number | 否 | 每页条数，默认20，范围1-100 |
| `offset` | number | 否 | 偏移量，默认0 |

**响应格式**：

```json
{
  "success": true,
  "data": {
    "transactions": [
      {
        "id": 1,
        "customerId": 480003,
        "type": "recharge",
        "amount": "1000.00",
        "balanceBefore": "0.00",
        "balanceAfter": "1000.00",
        "orderId": null,
        "orderNo": null,
        "notes": "管理员充值",
        "operatorId": 1,
        "operatorName": "管理员",
        "createdAt": "2026-02-06T10:00:00.000Z"
      },
      {
        "id": 2,
        "customerId": 480003,
        "type": "consume",
        "amount": "300.00",
        "balanceBefore": "1000.00",
        "balanceAfter": "700.00",
        "orderId": 123,
        "orderNo": "20260206085420-000",
        "notes": null,
        "operatorId": null,
        "operatorName": "用户自助下单",
        "createdAt": "2026-02-06T11:00:00.000Z"
      }
    ],
    "total": 2
  }
}
```

**SDK调用**：

```typescript
// 默认获取前20条
const result = await api.account.getMyTransactions();

// 分页获取
const page2 = await api.account.getMyTransactions({ limit: 10, offset: 10 });
```

### 2.3 查询指定客户余额（管理员/销售用）

**接口路径**：`account.getCustomerBalance`（tRPC query）

**认证要求**：需要登录（Token）

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `customerId` | number | 是 | 业务客户ID |

**响应格式**：

成功：
```json
{
  "success": true,
  "data": {
    "balance": "800.00",
    "customerId": 480003,
    "customerName": "张三"
  }
}
```

客户不存在：
```json
{
  "success": false,
  "error": "客户不存在"
}
```

**SDK调用**：

```typescript
const result = await api.account.getCustomerBalance(480003);
if (result.success) {
  console.log('客户余额:', result.data.balance);
} else {
  console.error(result.error);
}
```

### 2.4 查询指定客户流水（管理员/销售用）

**接口路径**：`account.getCustomerTransactions`（tRPC query）

### 2.5 客户充值（管理员/销售操作）

**接口路径**：`account.recharge`（tRPC mutation）

**认证要求**：需要登录（Token）

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `customerId` | number | 是 | 业务客户ID |
| `amount` | number | 是 | 充值金额，必须大于0 |
| `notes` | string | 否 | 备注 |

**响应格式**：

成功：
```json
{
  "success": true,
  "data": {
    "balanceBefore": "500.00",
    "balanceAfter": "800.00"
  }
}
```

失败：
```json
{
  "success": false,
  "error": "充值失败: 客户不存在"
}
```

**SDK调用**：

```typescript
const result = await api.account.recharge(480003, 300, '管理员充值');
if (result.success) {
  console.log(`充值成功: ${result.data.balanceBefore} → ${result.data.balanceAfter}`);
}
```

### 2.6 客户退款（管理员操作）

**接口路径**：`account.refund`（tRPC mutation）

**认证要求**：需要登录（Token）

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `customerId` | number | 是 | 业务客户ID |
| `amount` | number | 是 | 退款金额，必须大于0 |
| `orderId` | number | 是 | 关联订单ID |
| `orderNo` | string | 是 | 关联订单号 |

**响应格式**：

成功：
```json
{
  "success": true,
  "data": {
    "balanceBefore": "700.00",
    "balanceAfter": "1000.00"
  }

```

失败：
```json

}
```

## 三、业务流程说明

### 3.1 余额查询优先级

系统优先从 `accountTransactions` 流水表获取最新的 `balanceAfter` 作为当前余额。如果流水表为空，则取 `customers.accountBalance` 字段值。

### 3.2 下单余额扣除

当App用户通过 `orders.userCreate` 下单时，如果支付方式为"余额支付"，系统会自动调用 `consumeCustomerAccount` 扣除对应金额。扣款失败（如余额不足）会导致下单失败。

### 3.3 充值-消费-退款完整流程

```
充值1000 → 余额: 0.00 → 1000.00 (type: recharge)
消费300  → 余额: 1000.00 → 700.00 (type: consume)
退款100  → 余额: 700.00 → 800.00 (type: refund)
```

每一步操作都会在 `accountTransactions` 表中生成一条流水记录，包含操作前后余额。

### 3.4 金额格式

所有金额字段均为**字符串类型**，保留两位小数（如 `"800.00"`）。前端显示时无需额外格式化。

## 四、React Native 集成示例

### 4.1 余额页面

```typescript
import { api } from '../services/api';

// 查询余额
const fetchBalance = async () => {
  try {
    const result = await api.account.getMyBalance();
    if (result.success) {
      setBalance(result.data.balance);
      setCustomerName(result.data.customerName);
    }
  } catch (error) {
    console.error('查询余额失败:', error);
  }
};

// 查询流水（带分页）
const fetchTransactions = async (page: number) => {
  try {
    const limit = 20;
    const offset = page * limit;
    const result = await api.account.getMyTransactions({ limit, offset });
    if (result.success) {
      setTransactions(prev => [...prev, ...result.data.transactions]);
      setHasMore(result.data.total > offset + limit);
    }
  } catch (error) {
    console.error('查询流水失败:', error);
  }
};
```

## 五、错误码说明

| 错误信息 | 触发条件 | 建议处理 |
|----------|----------|----------|
| `客户不存在` | customerId无效 | 提示用户联系客服 |
| `充值失败: ...` | 充值过程异常 | 重试或联系管理员 |
| `退款失败: ...` | 退款过程异常 | 检查订单号是否正确 |
| `余额不足` | 消费金额超过余额 | 提示用户充值 |
| `UNAUTHORIZED` | Token无效或过期 | 跳转登录页 |
