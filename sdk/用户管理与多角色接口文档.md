# 用户管理与多角色系统 - 前端App接口文档

> 本文档面向前端App开发团队，涵盖用户认证（注册、登录、修改密码、忘记密码、Token刷新）和多角色系统的完整接口说明。所有接口已在后端通过单元测试验证，前端可直接调用。

---

## 一、角色体系说明

系统支持 **5种用户角色**，同一用户可拥有多个角色（多选）。角色以逗号分隔的字符串存储在 `roles` 字段中。

| 角色值 | 中文名称 | 说明 |
|--------|----------|------|
| `admin` | 管理员 | 系统超级管理员，拥有所有权限 |
| `teacher` | 老师 | 授课老师 |
| `user` | 普通用户 | App注册的普通用户（默认角色） |
| `sales` | 销售 | 销售人员 |
| `cityPartner` | 城市合伙人 | 城市合伙人 |

**多角色示例**：一个用户既是老师又是管理员，其 `roles` 字段值为 `"admin,teacher"`。

### 角色解析工具函数（前端可直接使用）

```typescript
// 角色常量
const USER_ROLES = ["admin", "teacher", "user", "sales", "cityPartner"] as const;
type UserRole = typeof USER_ROLES[number];

const ROLE_LABELS: Record<UserRole, string> = {
  admin: "管理员",
  teacher: "老师",
  user: "普通用户",
  sales: "销售",
  cityPartner: "城市合伙人",
};

// 解析角色字符串为数组
function parseRoles(rolesStr: string | null | undefined): UserRole[] {
  if (!rolesStr) return ["user"];
  return rolesStr.split(",").filter((r): r is UserRole =>
    USER_ROLES.includes(r as UserRole)
  );
}

// 检查用户是否拥有某个角色
function hasRole(rolesStr: string | null | undefined, role: UserRole): boolean {
  return parseRoles(rolesStr).includes(role);
}

// 检查用户是否拥有任一角色
function hasAnyRole(rolesStr: string | null | undefined, checkRoles: UserRole[]): boolean {
  const roles = parseRoles(rolesStr);
  return checkRoles.some(r => roles.includes(r));
}
```

## 二、认证接口

### 2.1 用户注册 `auth.register`

新用户通过手机号和密码注册账号。注册成功后自动返回JWT Token，可直接登录。注册时自动创建业务客户记录。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `phone` | string | 是 | 手机号（11位） |
| `password` | string | 是 | 密码（6-20位） |
| `name` | string | 否 | 用户昵称（不传则默认为手机号） |

**响应格式**：

```typescript
interface RegisterResult {
  success: boolean;
  message: string;       // 成功/失败消息
  token?: string;        // JWT Token（注册成功时返回）
  user?: {
    id: number;
    openId: string;
    phone: string;
    name: string;
    role: string;        // 兼容旧字段，单角色
    roles: string;       // 多角色，逗号分隔，如 "user"
  };
}
```

**调用示例**：

```typescript
const result = await api.auth.register({
  phone: '15921456877',
  password: '123456',
  name: '张三'
});

if (result.success) {
  // 注册成功，保存Token
  await AsyncStorage.setItem('token', result.token!);
  // 解析用户角色
  const roles = parseRoles(result.user!.roles); // ["user"]
  navigation.navigate('Home');
} else {
  Alert.alert('注册失败', result.message);
}
```

**错误场景**：

| 错误信息 | 说明 |
|----------|------|
| `该手机号已被注册` | 手机号已存在 |
| `手机号格式不正确` | 非11位数字 |
| `密码长度必须在6-20位之间` | 密码不符合要求 |

---

### 2.2 用户登录 `auth.loginWithUserAccount`

通过用户名/手机号 + 密码登录。登录时自动检查并补充业务客户记录。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `username` | string | 是 | 用户名或手机号 |
| `password` | string | 是 | 密码 |

**响应格式**：

```typescript
interface LoginResult {
  success: boolean;
  token?: string;        // JWT Token
  user?: {
    id: number;
    openId: string;
    name: string;
    role: string;        // 兼容旧字段
    roles: string;       // 多角色，如 "admin,teacher"
  };
  error?: string;        // 错误信息
}
```

**调用示例**：

```typescript
const result = await api.auth.login({
  username: '15921456877',
  password: '123456'
});

if (result.success) {
  await AsyncStorage.setItem('token', result.token!);
  
  // 根据角色跳转不同页面
  const roles = parseRoles(result.user!.roles);
  if (hasRole(result.user!.roles, 'admin')) {
    navigation.navigate('AdminDashboard');
  } else if (hasRole(result.user!.roles, 'teacher')) {
    navigation.navigate('TeacherDashboard');
  } else {
    navigation.navigate('Home');
  }
} else {
  Alert.alert('登录失败', result.error || '用户名或密码错误');
}
```

**错误场景**：

| error值 | 说明 |
|---------|------|
| `用户不存在` | 用户名/手机号未注册 |
| `密码错误` | 密码不匹配 |
| `账号已被禁用` | 账号被管理员禁用 |

---

### 2.3 Token刷新 `auth.refreshToken`

JWT Token有效期为7天，建议在Token过期前主动刷新。SDK内置了自动刷新机制。

**请求参数**：无（通过当前Token自动识别用户）

**响应格式**：

```typescript
interface RefreshTokenResult {
  success: boolean;
  token?: string;        // 新的JWT Token
  expiresIn?: number;    // 有效期（秒），默认604800（7天）
}
```

**调用示例**：

```typescript
// SDK内置自动刷新，通常不需要手动调用
// 如需手动刷新：
const result = await api.auth.refreshToken();
if (result.success) {
  await AsyncStorage.setItem('token', result.token!);
}
```

### 2.4 修改密码 `auth.changePassword`

**需要登录**（通过Token自动获取用户身份，不需要传userId）。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `oldPassword` | string | 是 | 旧密码 |
| `newPassword` | string | 是 | 新密码（6-20位） |

**响应格式**：

```typescript
interface ChangePasswordResult {
  success: boolean;
  error?: string;        // 错误信息（可选）
}
```

**调用示例**：

```typescript
const result = await api.auth.changePassword({
  oldPassword: '123456',
  newPassword: 'newPass789'
});

if (result.success) {
  // 修改成功，清除Token，跳转登录页
  await AsyncStorage.removeItem('token');
  Alert.alert('成功', '密码修改成功，请重新登录');
  navigation.navigate('Login');
} else {
  Alert.alert('修改失败', result.error || '未知错误');
}
```

**错误场景**：

| error值 | 说明 |
|---------|------|
| `旧密码错误` | 旧密码验证失败 |
| `新密码不能与旧密码相同` | 新旧密码一致 |
| `新密码长度必须在6-20位之间` | 密码长度不符合要求 |
| `该账号未设置密码，无法修改` | 用户没有设置过密码 |

---

### 2.5 忘记密码（重置密码） `auth.resetPassword`

**不需要登录**。通过手机号 + 验证码重置密码。

> **测试环境**：验证码固定为 `123456`，无需真实发送短信。生产环境将接入短信服务。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `phone` | string | 是 | 手机号（11位） |
| `code` | string | 是 | 短信验证码（测试环境固定 `123456`） |
| `newPassword` | string | 是 | 新密码（6-20位） |

**响应格式**：

```typescript
interface ResetPasswordResult {
  success: boolean;
  error?: string;        // 错误信息（可选）
}
```

**调用示例**：

```typescript
const result = await api.auth.resetPassword({
  phone: '15921456877',
  code: '123456',        // 测试环境固定值
  newPassword: 'newPass789'
});

if (result.success) {
  Alert.alert('成功', '密码重置成功，请使用新密码登录');
  navigation.navigate('Login');
} else {
  Alert.alert('重置失败', result.error || '未知错误');
}
```

**错误场景**：

| error值 | 说明 |
|---------|------|
| `该手机号未注册` | 手机号不存在 |
| `验证码错误` | 验证码不匹配 |
| `手机号格式不正确` | 非11位数字 |
| `新密码长度必须在6-20位之间` | 密码长度不符合要求 |

## 三、用户信息与角色

### 3.1 获取当前用户信息

登录和注册接口返回的 `user` 对象中包含 `roles` 字段。建议在App启动时通过 `refreshToken` 获取最新的用户信息。

```typescript
// 登录后保存用户信息
const loginResult = await api.auth.login({ username, password });
if (loginResult.success) {
  const user = loginResult.user!;
  
  // 保存用户信息
  await AsyncStorage.setItem('token', loginResult.token!);
  await AsyncStorage.setItem('user', JSON.stringify(user));
  
  // 解析角色
  const roles = parseRoles(user.roles);
  console.log('用户角色:', roles);
  // 例如: ["admin", "teacher"]
  
  // 获取角色中文名
  const roleNames = roles.map(r => ROLE_LABELS[r]);
  console.log('角色名称:', roleNames);
  // 例如: ["管理员", "老师"]
}
```

### 3.2 角色判断与页面权限控制

开发阶段不限制页面权限，但前端可提前做好角色判断的基础架构：

```typescript
// 在React Native中使用角色判断
function useUserRoles() {
  const [user, setUser] = useState(null);
  
  // 从AsyncStorage加载用户信息
  useEffect(() => {
    AsyncStorage.getItem('user').then(data => {
      if (data) setUser(JSON.parse(data));
    });
  }, []);
  
  return {
    user,
    roles: user ? parseRoles(user.roles) : [],
    isAdmin: user ? hasRole(user.roles, 'admin') : false,
    isTeacher: user ? hasRole(user.roles, 'teacher') : false,
    isSales: user ? hasRole(user.roles, 'sales') : false,
    isCityPartner: user ? hasRole(user.roles, 'cityPartner') : false,
    isUser: user ? hasRole(user.roles, 'user') : false,
  };
}

// 在页面中使用
function ProfileScreen() {
  const { user, roles, isAdmin, isTeacher } = useUserRoles();
  
  return (
    <View>
      <Text>用户: {user?.name}</Text>
      <Text>角色: {roles.map(r => ROLE_LABELS[r]).join('、')}</Text>
      
      {isAdmin && <AdminPanel />}
      {isTeacher && <TeacherSchedule />}
    </View>
  );
}
```

## 五、接口汇总表

| 接口 | SDK方法 | 需要登录 | 说明 |
|------|---------|----------|------|
| `auth.register` | `api.auth.register()` | 否 | 手机号+密码注册，返回Token |
| `auth.loginWithUserAccount` | `api.auth.login()` | 否 | 用户名/手机号+密码登录 |
| `auth.refreshToken` | `api.auth.refreshToken()` | 是 | 刷新JWT Token |

## 六、注意事项

1. **Token有效期**：JWT Token有效期为7天（604800秒），建议在App启动时调用 `refreshToken` 检查并刷新Token。

2. **roles字段兼容性**：接口同时返回 `role`（单角色，兼容旧版本）和 `roles`（多角色，逗号分隔）。前端应优先使用 `roles` 字段。

3. **默认角色**：新注册用户默认角色为 `user`（普通用户），管理员可在后台为用户分配多个角色。

4. **验证码**：当前测试环境验证码固定为 `123456`，生产环境将接入短信服务后替换。

5. **密码规则**：密码长度6-20位，支持字母、数字、特殊字符组合。

6. **错误处理统一格式**：所有接口错误返回格式为 `{ success: false, error: "错误信息" }`，前端统一通过 `error` 字段获取错误信息。
