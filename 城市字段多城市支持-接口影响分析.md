# 城市字段多城市支持 - 接口影响分析

**修改日期**: 2026年2月3日  
**功能**: 支持老师关联多个城市(分号分隔)

---

## 📋 改动内容

### 1. 数据库层面

**无需修改** ✅

- `teachers.city` 字段类型: `varchar(50)`
- 现有字段长度足够存储多城市字符串
- 示例: "北京;上海;深圳" (11个字符)

### 2. 后端API层面

#### 2.1 teachers.create (新增老师)

**改动**: 城市字段改为必填,添加格式验证

```typescript
city: z.string().min(1, "城市不能为空").refine(
  (val) => {
    // 验证城市格式:支持单个城市或多个城市(分号分隔)
    const cities = val.split(';').map(c => c.trim()).filter(c => c !== '');
    return cities.length > 0;
  },
  { message: "请输入有效的城市名称,多个城市用分号分隔" }
)
```

**影响**: 
- ✅ 向后兼容 - 单城市字符串仍然有效
- ✅ 新增验证 - 防止空字符串或无效格式
- ⚠️ **破坏性变更** - 城市字段从可选变为必填

#### 2.2 teachers.update (更新老师)

**改动**: 添加城市格式验证(仍然可选)

```typescript
city: z.string().optional().refine(
  (val) => {
    // 如果提供了city值,验证格式
    if (!val || val.trim() === '') return true; // 空值允许(编辑时可选)
    const cities = val.split(';').map(c => c.trim()).filter(c => c !== '');
    return cities.length > 0;
  },
  { message: "请输入有效的城市名称,多个城市用分号分隔" }
)
```

**影响**:
- ✅ 向后兼容 - 不修改城市时不影响
- ✅ 格式验证 - 防止输入无效格式

#### 2.3 teachers.list (查询老师列表)

**无需修改** ✅

```typescript
list: publicProcedure.query(async () => {
  return db.getAllTeachers();
})
```

**返回数据示例**:

```json
[
  {
    "id": 120001,
    "name": "安雅",
    "customerType": "温柔御姐",
    "notes": "复购高,态度好",
    "city": "北京;上海;深圳",  // 多城市字符串
    "isActive": true
  }
]
```

**影响**:
- ✅ 完全兼容 - city字段仍然是字符串类型
- ✅ 前端需要处理 - 如果需要显示多个城市,前端需要split处理

#### 2.4 teachers.getById (查询单个老师)

**无需修改** ✅

**影响**:
- ✅ 完全兼容 - 返回完整的老师信息,包括多城市字符串

### 2. 课程预约App (前端App)

#### 2.1 老师选择器 (teacher-selector.tsx)

**当前过滤逻辑**:

```typescript
teachers.filter(t => t.isActive && t.city === selectedCity)
```

**影响**:
- ⚠️ **需要修改** - 单城市匹配无法处理多城市字符串
- 🔧 **修复方案** - 改为包含匹配

**修复代码**:

```typescript
teachers.filter(t => {
  if (!t.isActive) return false;
  if (!t.city) return false;
  
  // 支持多城市匹配
  const cities = t.city.split(';').map(c => c.trim());
  return cities.includes(selectedCity);
})
```
